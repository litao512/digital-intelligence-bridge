# 插件启动检查体验设计

## 背景

当前客户端启动后会执行插件完整更新流程：检查、下载、预安装。这样会导致用户打开插件中心时直接看到“重启生效”，与“每个插件独立下载、用户主动安装/更新”的新体验冲突。

## 目标

启动时只检查插件更新，不下载插件包，不生成预安装目录。若存在可安装或可更新插件，使用非打扰提示引导用户进入插件中心，由用户手动点击“安装”或“更新”后才开始下载。

## 方案

采用“静默检查 + 状态提示 + 手动下载”的方案：

- 启动阶段调用插件更新编排器的 `CheckAsync(Startup)`，不调用 `RunAsync(Startup)`。
- 检查结果中的授权插件详情同步给插件中心，使插件中心打开时能显示“可安装”或“可更新”。
- 若检查到插件更新，更新托盘提示文本为“DIB客户端 · N 个插件可更新”，并记录托盘通知日志。
- 插件中心仍然是唯一触发插件包下载和预安装的主入口。

## 不做的事

- 不弹模态框。
- 不在启动阶段下载插件包。
- 不在启动阶段生成 staging 预安装目录。
- 暂不改造托盘菜单项文本。现有 `ITrayService` 只支持添加/删除菜单项，不支持安全更新已有菜单项文本；此项可后续作为托盘菜单增强单独处理。

## 验收标准

- 启动检查只调用 `CheckAsync(Startup)`。
- 启动检查不会调用 `RunAsync`，因此不会自动下载或预安装插件。
- 检查到授权插件后，插件中心打开时显示“安装”或“更新”。
- 有可用插件时，托盘提示会显示可更新数量。
- 既有插件中心手动安装、更新、停止下载、重启生效流程不回退。
