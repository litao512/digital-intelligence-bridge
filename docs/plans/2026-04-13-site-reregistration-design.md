# 托盘重新注册设计

**目标**

在系统托盘增加“重新注册站点”入口，允许用户快速修正站点资料；站点资料补充“使用单位”，并纳入站点身份导入导出。

**设计结论**

采用轻量独立对话框，而不是跳转到设置页。托盘菜单新增“重新注册站点”，点击后弹出小窗口，字段为 `使用单位`、`站点名称`、`站点备注`。保存时只更新站点资料，不重置 `SiteId`。

**数据范围**

- 本地配置 `ReleaseCenter` 新增 `SiteOrganization`
- 站点身份文件 `site-identity.json` 新增 `SiteOrganization`
- 发布中心注册展示名升级为 `使用单位 / 站点名称`

**校验**

- `使用单位` 必填
- `站点名称` 必填
- `站点备注` 选填

**兼容性**

站点身份文件导出版本升级；导入时兼容旧版本，无“使用单位”的旧文件仍可导入，但导入后界面会保留该字段为空，便于用户补全。
