# 客户端升级助手设计

## 目标

补齐客户端升级闭环：客户端下载并校验升级包后，用户可点击“立即升级”，由独立升级助手等待主程序退出、解压覆盖客户端目录，并重新启动新版客户端。

## 方案

- 新增 `DibClient.Updater` 控制台程序，负责执行文件覆盖和重启。
- 主客户端新增 `ClientUpgradeService`，负责校验升级包路径、定位升级助手、复制升级助手到临时目录、启动升级助手，并退出当前客户端。
- 设置页在客户端下载成功后显示“立即升级”按钮；按钮不可用时不打扰用户。
- 升级助手运行参数包括：升级包路径、目标目录、主程序 exe 路径、待等待的进程 ID、是否重启。
- 为避免 Windows 文件锁，升级助手从 `%LOCALAPPDATA%\DibClient\updater-runs\...` 临时目录运行，不从被覆盖的客户端目录运行。

## 风险与边界

- 当前实现面向 portable zip 包，不做 MSI/安装器能力。
- 覆盖失败时写入日志并返回非零退出码；不会删除升级包。
- 目标目录必须是当前主程序所在目录，避免误覆盖其他路径。
