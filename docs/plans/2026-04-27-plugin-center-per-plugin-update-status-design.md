# 插件中心逐插件更新状态设计

## 背景

插件中心第一版已经提供独立入口、插件列表和基础生命周期操作，但它仍主要展示本地运行时目录和预安装目录的状态。随着插件数量增加，用户需要看到每个插件的当前版本、最新版本和更新状态，而不是只看到整体更新摘要。

本设计在现有插件中心上扩展“逐插件更新状态”，让插件中心成为多插件场景下的主要管理视图。

## 目标

1. 插件中心列表显示当前版本、最新版本和逐插件状态。
2. 点击“检查更新”后，插件中心刷新发布中心授权插件明细，并与本地插件状态合并。
3. 支持区分已最新、可更新、未安装、待重启生效和更新失败。
4. 保持现有自动下载和预安装策略，不增加单插件更新按钮。
5. 不改变发布中心 manifest 协议。

## 非目标

1. 不做单插件手动升级。
2. 不做启用、禁用、卸载插件。
3. 不做后台定时轮询。
4. 不做插件热加载。
5. 不做插件二级菜单折叠。

## 状态定义

插件中心逐项状态按优先级判断：

1. `待重启生效`：预安装目录中存在该插件版本。
2. `未安装`：发布中心授权插件中存在，但本地运行时目录不存在。
3. `可更新`：发布中心授权插件版本高于本地运行时版本。
4. `已最新`：发布中心授权插件版本等于本地运行时版本。
5. `已生效`：本地存在插件，但本次没有获取到发布中心授权版本。
6. `更新失败`：最近一次更新流程失败，且失败详情能关联到插件时显示；第一版无法关联到具体插件时，保留整体失败详情。

版本比较采用语义化版本（Semantic Versioning）优先；如果无法解析，退化为忽略大小写的字符串比较。不阻塞页面显示。

## 数据来源

### 本地运行时插件

读取 runtime 插件目录下的 `plugin.json`：

- `id`
- `name`
- `version`

状态基础值为 `已生效`。

### 预安装插件

读取 staging 插件目录下的 `plugin.json`：

- `id`
- `name`
- `version`

状态基础值为 `待重启生效`，优先覆盖运行时状态。

### 发布中心授权插件

复用 `ReleaseCenterCheckResult.AuthorizedPluginDetail`。当前格式为每行：

```text
插件名称 / 插件 ID / 版本
```

插件中心在检查更新后解析该明细，作为“最新版本”来源。

## 用户界面

插件中心列表列调整为：

- 插件名称
- 插件 ID
- 当前版本
- 最新版本
- 状态
- 详情

总览卡片调整为：

- 已最新
- 可更新
- 待重启
- 异常

“检查更新”按钮继续执行自动检查、下载和预安装流程。流程结束后刷新逐插件状态。

## 数据模型

`PluginCenterItem` 增加：

- `CurrentVersion`
- `LatestVersion`
- `Detail`

保留 `Version` 可作为兼容别名，返回 `CurrentVersion` 或待重启版本。

`PluginCenterViewModel` 增加最近一次授权插件缓存：

- `IReadOnlyList<PluginCenterAvailablePlugin> _availablePlugins`

检查更新成功后，从 `PluginUpdateRunResult.CheckResult.AuthorizedPluginDetail` 解析并更新缓存。

## 错误处理

1. 发布中心检查失败：保留本地插件列表，显示整体失败详情。
2. 授权插件明细为空：本地插件状态显示为 `已生效`。
3. 明细格式异常：跳过异常行，不影响其他插件。
4. 版本解析失败：使用字符串比较；仍无法判断时显示 `已生效`，详情说明无法比较版本。

## 测试策略

1. 本地版本低于授权版本时显示 `可更新`。
2. 本地不存在但授权存在时显示 `未安装`。
3. staging 版本存在时显示 `待重启生效`，并显示最新版本。
4. 本地版本等于授权版本时显示 `已最新`。
5. 检查更新后能解析 `AuthorizedPluginDetail` 并刷新列表。
6. XAML 表格包含当前版本和最新版本列。

## 验收标准

1. 插件中心能逐项显示当前版本和最新版本。
2. 多个插件可以分别显示不同状态。
3. 点击“检查更新”后，状态能根据发布中心返回的授权插件明细刷新。
4. 自动下载和预安装策略保持不变。
5. 客户端构建和相关单元测试通过。
